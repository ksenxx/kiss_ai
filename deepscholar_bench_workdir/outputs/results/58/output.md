# Related Work

**Keyword-based and semantic table retrieval.** Early ad hoc table search treats a table as a bag of text and ranks candidates against a small set of user keywords. Zhang and Balog introduced a semantic-similarity framework that combines lexical and embedding-based signals for table retrieval [1](https://arxiv.org/abs/1802.06159), and later unified keyword and table queries into a single retrieval model [2](https://arxiv.org/abs/2105.06365). Table2Vec learns joint word and entity embeddings tailored for table population and retrieval [3](https://arxiv.org/abs/1906.00041). Neural rankers subsequently replaced sparse retrieval: Chen et al. proposed a deep contextualized language model for table search [4](https://arxiv.org/abs/2005.09207), StruBERT injects table structure into a BERT-style encoder for search and matching [5](https://arxiv.org/abs/2203.14278), and Wang et al. cast table retrieval as multi-granular graph representation learning to handle heterogeneous structures [6](https://arxiv.org/abs/2105.01736). While these methods enrich the query with semantics, they still assume unstructured keywords or short natural-language questions rather than the mixed table-plus-NL input that nlcTD targets.

**Query-table-based discovery: union and join search.** A second line of work retrieves tables that are unionable or joinable with a user-supplied query table. Bogatu et al. formalized dataset discovery in data lakes with attribute-level relatedness measures [7](https://arxiv.org/abs/2011.10427). SANTOS exploits column relationships and semantic types for table union search [8](https://arxiv.org/abs/2209.13589), while Starmie learns contextualized column representations to unify union and related-table discovery [9](https://arxiv.org/abs/2210.01922). For joinable-table discovery, Dong et al. proposed a high-dimensional similarity-based method to scale joinability search in data lakes [10](https://arxiv.org/abs/2010.13273), and their follow-up DeepJoin leverages pre-trained language models to represent columns for joinable retrieval [11](https://arxiv.org/abs/2212.07588). TURL provides a foundational pre-training paradigm for table understanding that supports many downstream discovery tasks [12](https://arxiv.org/abs/2006.14806). These systems rank candidate tables purely from a query table and cannot incorporate the user's textual intent, which is central to our nlcTD formulation.

**Corpora and benchmarks for table tasks.** Progress in this area is enabled by large, structured table corpora. GitTables provides a million-scale corpus of relational tables mined from GitHub and is widely used to stress-test discovery pipelines [13](https://arxiv.org/abs/2106.07258). SANTOS also releases benchmark queries for semantic union search [8](https://arxiv.org/abs/2209.13589). None of these resources, however, pair a query table with a natural-language condition to jointly test relevance across a query table, an NL requirement, and a candidate table, which motivates the 627 queries, 22,080 candidate tables, and 21,200 relevance labels released with nlcTables.

**LLMs, RAG, and natural-language interfaces over tables.** With the rise of LLMs, natural-language querying of tabular data has moved beyond retrieval into generation. Solo pursues data discovery driven by natural-language questions through a self-supervised approach [14](https://arxiv.org/abs/2301.03560), and TableRAG couples retrieval with LLM reasoning to enable million-token table understanding [15](https://arxiv.org/abs/2410.04739). The BIRD benchmark evaluates LLMs as database interfaces for large-scale text-to-SQL [16](https://arxiv.org/abs/2305.03111), and a recent survey positions generative AI as a key enabler for tabular data augmentation [17](https://arxiv.org/abs/2407.21523). These works underline the demand for expressive NL-driven access to tabular data, but they focus on question answering or SQL synthesis over given tables, not on ranking candidate tables that satisfy a joint table-plus-NL request.

**Foundational building blocks.** Our baselines rely on standard components including distilled Transformer encoders such as DistilBERT [18](https://arxiv.org/abs/1910.01108), permuted-masked pre-training objectives such as MPNet [19](https://arxiv.org/abs/2004.09297), and HNSW graphs for approximate nearest-neighbor search over dense embeddings [20](https://arxiv.org/abs/1603.09320).
