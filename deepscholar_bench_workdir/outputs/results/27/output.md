# Related Work

**Natural-Language Table Discovery.** Discovering relevant tables from large repositories has traditionally been formulated as unionable or joinable table search. Systems such as D3L cast the problem as attribute-level similarity retrieval over heterogeneous features [1](https://arxiv.org/abs/2011.10427), while SANTOS enriches the signal with binary relationships between columns to capture table semantics [2](https://arxiv.org/abs/2209.13589). Starmie further pushes this direction by learning contextualized column embeddings through contrastive pre-training over data-lake tables [3](https://arxiv.org/abs/2210.01922). LakeBench provides a comprehensive benchmark that compares these methods on realistic data-lake workloads [4](https://arxiv.org/abs/2307.04217), and a recent critical re-evaluation questions the robustness of current table-union benchmarks and the methods evaluated on them [5](https://arxiv.org/abs/2505.21329). All of these systems follow a representation–index–search pipeline that Birdie explicitly seeks to replace.

**Table Representation Learning.** A parallel line of work pretrains Transformer encoders over tabular data to support downstream understanding tasks. TAPAS injects row/column embeddings into BERT for weakly supervised table parsing [6](https://arxiv.org/abs/2004.02349), TaBERT jointly encodes natural-language utterances and table content snapshots [7](https://arxiv.org/abs/2005.08314), TURL introduces visibility-aware masked-column pretraining for relational tables [8](https://arxiv.org/abs/2006.14806), and TABBIE learns dual row/column transformer views over cell tokens [9](https://arxiv.org/abs/2105.02584). Birdie leverages such encoders to serialize tables but couples them with a generative decoder that directly emits table identifiers.

**Dense Retrieval.** State-of-the-art NL-driven table discovery adapts dense passage retrievers to the table setting. DPR established dual-encoder training for open-domain question answering [10](https://arxiv.org/abs/2004.04906), ColBERT introduced late interaction to preserve token-level matching signals [11](https://arxiv.org/abs/2004.12832), ANCE improved training via approximate-nearest-neighbor hard-negative mining [12](https://arxiv.org/abs/2007.00808), Contriever showed that unsupervised contrastive pretraining transfers well across domains [13](https://arxiv.org/abs/2112.09118), GTR scaled dual encoders for generalizable retrieval [14](https://arxiv.org/abs/2112.07899), and E5 exploited weakly supervised text pairs to obtain broadly competitive embeddings [15](https://arxiv.org/abs/2212.03533). These pipelines suffer from cascaded errors between representation and indexing that Birdie aims to eliminate.

**Generative and Differentiable Search Indexes.** Birdie belongs to the emerging family of generative retrieval, which encodes the entire corpus in the parameters of a seq2seq model that directly decodes document identifiers. The Differentiable Search Index (DSI) first demonstrated this idea for passage retrieval [16](https://arxiv.org/abs/2202.06991); the Neural Corpus Indexer added semantic docid trees and prefix-aware decoding [17](https://arxiv.org/abs/2206.02743); SEAL generates arbitrary n-gram substrings as identifiers via FM-index constrained decoding [18](https://arxiv.org/abs/2204.10628); Ultron combines model-based indexing with URL/title-based identifiers [19](https://arxiv.org/abs/2208.09257); and GenRet learns discrete document tokenizations end-to-end for generative retrieval [20](https://arxiv.org/abs/2304.04171). Birdie extends this paradigm from passages to tables by designing prefix-aware table identifiers tailored to the structured, multi-modal nature of tabular data.

**Query Generation for Retrieval.** Bridging the vocabulary gap between queries and documents through synthetic query generation is a well-established technique. Doc2query/docT5query augments each document with predicted queries before indexing [21](https://arxiv.org/abs/1904.08375). InPars uses large language models to synthesize (query, document) pairs for few-shot retriever training [22](https://arxiv.org/abs/2202.05144), Promptagator prompts an LLM with only a handful of examples to bootstrap task-specific retrievers [23](https://arxiv.org/abs/2209.11755), InPars-v2 refines this pipeline with open LLMs and reranker filtering [24](https://arxiv.org/abs/2301.01820), and HyDE explores the dual direction of generating hypothetical documents from queries [25](https://arxiv.org/abs/2212.10496). DSI-QG in particular showed that generated queries can close the indexing–retrieval gap for differentiable search indexes [26](https://arxiv.org/abs/2206.10128). Birdie adapts this idea to tables by prompting an LLM to synthesize NL queries that reflect each table's content.

**Continual Indexing and Catastrophic Forgetting.** Because table repositories evolve, indexes must be updated without retraining from scratch. DSI++ studies how differentiable indexes can absorb new documents while retaining old ones [27](https://arxiv.org/abs/2212.09744), and a broader line of empirical studies documents catastrophic forgetting when fine-tuning large language models continually [28](https://arxiv.org/abs/2308.08747). Analyses of parameter-efficient tuning suggest that parameter isolation is an effective mitigation [29](https://arxiv.org/abs/2402.18865), and dedicated benchmarks for continual neural information retrieval quantify forgetting in retriever backbones [30](https://arxiv.org/abs/2308.08378). Birdie's parameter-isolation update strategy builds on these insights to keep prior table knowledge intact while incrementally indexing newly arriving tables.
