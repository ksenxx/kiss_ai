# Related Works

**Large Language Models and the Edge Deployment Challenge.** The rapid scaling of transformer-based large language models (LLMs), such as GPT-3 [1](https://arxiv.org/abs/2005.14165), LLaMA [2](https://arxiv.org/abs/2302.13971), and LLaMA 2 [3](https://arxiv.org/abs/2307.09288), has produced unprecedented natural language capabilities at the cost of massive compute, memory, and energy requirements. Broad surveys of efficient LLM techniques [4](https://arxiv.org/abs/2312.03863) and of resource-efficient foundation-model systems [5](https://arxiv.org/abs/2401.08092) systematically catalog the algorithmic and system-level barriers that make deploying such models on storage-, power-, and thermally-constrained edge devices exceptionally difficult. CLONE targets exactly this deployment gap through a co-design that couples model compression with a specialized 28 nm accelerator.

**Weight and Activation Quantization.** A dominant line of prior work reduces LLM footprint through low-precision numerics. LLM.int8() shows that emergent outlier features can be handled by mixed-precision decomposition to preserve accuracy at 8 bits [6](https://arxiv.org/abs/2208.07339). GPTQ generalizes second-order post-training quantization down to 3–4 bits for billion-parameter models [7](https://arxiv.org/abs/2210.17323), while SmoothQuant migrates activation outliers into weights to enable joint W8A8 quantization [8](https://arxiv.org/abs/2211.10438). AWQ [9](https://arxiv.org/abs/2306.00978) and SqueezeLLM [10](https://arxiv.org/abs/2306.07629) further push into 4-bit and non-uniform, dense-and-sparse regimes tailored to salient weights. GOBO exemplifies how quantization can be exploited by a dedicated NLP accelerator for latency and energy savings [11](https://arxiv.org/abs/2005.03842). CLONE builds on this literature but jointly optimizes quantization with hardware datapaths on always-on edge platforms rather than server GPUs.

**Pruning and Sparsity.** Complementing quantization, structured and unstructured pruning shrink model size and compute. SparseGPT enables one-shot 50%+ pruning of massive LLMs with negligible perplexity loss [12](https://arxiv.org/abs/2301.00774), and Wanda proposes a simple activation-aware magnitude criterion that requires no retraining [13](https://arxiv.org/abs/2306.11695). Beyond static pruning, contextual sparsity approaches such as Deja Vu dynamically skip attention heads and MLP neurons at inference time [14](https://arxiv.org/abs/2310.17157), while H2O maintains only a small heavy-hitter subset of the KV cache [15](https://arxiv.org/abs/2306.14048). These dynamic sparsity patterns motivate CLONE's real-time energy-optimization pathway, but they are typically evaluated on GPU systems and rarely address the memory-hierarchy realities of edge SoCs.

**Efficient Attention and Serving Systems.** Attention is a primary latency and energy bottleneck for LLM inference. FlashAttention reorganizes the attention kernel to be IO-aware, avoiding materialization of the full attention matrix [16](https://arxiv.org/abs/2205.14135). PagedAttention/vLLM introduces virtual-memory-style KV-cache management for high-throughput serving [17](https://arxiv.org/abs/2309.06180), and StreamingLLM enables unbounded-length generation by preserving attention-sink tokens [18](https://arxiv.org/abs/2309.17453). Speculative decoding accelerates autoregressive generation by verifying draft-model tokens in parallel [19](https://arxiv.org/abs/2211.17192). These techniques inform, but do not solve, the edge constraints CLONE addresses.

**On-Device and Mobile LLM Inference.** A growing body of work targets constrained hardware directly. LLM in a Flash streams parameters from flash memory using sparsity- and locality-aware loading [20](https://arxiv.org/abs/2312.11514). PowerInfer exploits neuron-activation locality to combine a consumer GPU with CPU offloading [21](https://arxiv.org/abs/2312.12456), and EdgeMoE partitions mixture-of-experts LLMs across the mobile memory hierarchy [22](https://arxiv.org/abs/2308.14352). LLMCad co-schedules a small on-device generator with a large verifier for fast mobile inference [23](https://arxiv.org/abs/2309.04255), while MobileLLM redesigns sub-billion parameter architectures specifically for on-device use [24](https://arxiv.org/abs/2402.14905). These works stay largely at the software or commodity-hardware layer.

**Hardware Accelerators for Transformer Inference.** Finally, custom accelerators explicitly co-designed with attention algorithms provide a template for CLONE's silicon. A^3 approximates attention scoring to accelerate the dot-product bottleneck [25](https://arxiv.org/abs/2002.10941), and SpAtten introduces cascade token and head pruning tightly coupled to a dedicated architecture [26](https://arxiv.org/abs/2012.09852). CLONE extends this hardware-algorithm co-design paradigm to a 28 nm scalable edge accelerator that unifies quantization, sparsity, and dynamic energy optimization while preserving generality across always-on and intermediate edge workloads.
